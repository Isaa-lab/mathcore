fix(ai-chat): 根治 FUNCTION_INVOCATION_FAILED + 让"平台内置"真正零配置

从用户诊断详情里抓到的关键信号：返回的不是超时而是
"A server error has occurred / FUNCTION_INVOCATION_FAILED" ——
这说明 Vercel lambda 抛了未捕获异常，整个函数直接挂掉，
所以连我们写好的 diag 字段都没机会返回。

修复：

1) 后端：顶层 try/catch 守卫
   - 把整个 handler 拆成 runHandler，外层套 try/catch
   - 任何同步/异步异常都返回 { error, diag:"handler_crash", stack }
   - Vercel 再也不会返回裸 HTML，前端一定能拿到 JSON 定位

2) 前端：识别 backend_crash
   - classifyChatError 新增 "backend_crash" 分类
   - 识别 FUNCTION_INVOCATION_FAILED / A server error / handler_crash
   - 错误气泡提供"再试一次"+"复制诊断"按钮（方便反馈）

3) 前端：no_key 错误文案改得更诚实
   - 老版本说"去 AI 设置加 Key"，但用户往往已经点了「平台内置」
   - 新版本直接点破"平台 GROQ_KEY 没跑通"，给出两条路：
     · 如果你是管理员 → 去 Vercel 配 GROQ_KEY 环境变量
     · 否则 → 自己填一个 Key

4) 前端：强化「平台内置」的零门槛信号
   - desc 改为"点一下就能用·无需 Key·平台承担成本"
   - 加"推荐"徽章（覆盖原来的"免费"徽章）
   - 保持列表第一位，用绿色强调
